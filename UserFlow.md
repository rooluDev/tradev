## 1. 전체 서비스 맵

```
Tradev
├── 공개 영역 (비로그인)
│   ├── 랜딩 페이지          /
│   ├── 상품 목록            /items
│   ├── 상품 상세            /items/:id
│   ├── 로그인               /login
│   └── 회원가입             /signup
│
├── 사용자 영역 (로그인 필수)
│   ├── 상품 등록/수정        /items/new, /items/:id/edit
│   ├── 채팅 목록/상세        /chats, /chats/:id
│   ├── 거래 내역            /trades
│   ├── 알림                 /notifications
│   └── 마이페이지
│       ├── 프로필           /profile/:userId
│       ├── 프로필 수정      /profile/edit
│       ├── 관심 상품        /profile/wishlist
│       └── 예약 슬롯 관리   /profile/slots
│
└── 관리자 영역 (Admin 전용, MPA)
    ├── 대시보드             /admin
    ├── 회원 관리            /admin/users
    ├── 상품 관리            /admin/items
    ├── 신고 처리            /admin/reports
    ├── 카테고리 관리        /admin/categories
    └── 거래 통계            /admin/stats
```

## 2. 인증 플로우

### 2.1 이메일 회원가입

```
[랜딩 / 로그인 페이지]
        │
        │ "회원가입" 클릭
        ▼
[회원가입 페이지 /signup]
  - 이메일 입력
  - 비밀번호 / 확인 입력
  - 닉네임 입력
        │
        │ 실시간 유효성 검사
        │   ├── 이메일 형식 오류 → 인라인 에러 표시
        │   ├── 이메일 중복 (API 호출) → "이미 사용 중인 이메일"
        │   ├── 비밀번호 불일치 → 인라인 에러 표시
        │   └── 닉네임 중복 (API 호출) → "이미 사용 중인 닉네임"
        │
        │ "가입하기" 클릭 (유효성 통과)
        ▼
[이메일 인증 안내 페이지]
  - "인증 메일이 발송되었습니다" 안내
  - 재발송 버튼 (60초 쿨다운)
        │
        │ 사용자가 메일 링크 클릭
        ▼
[이메일 인증 완료]
  - 인증 성공 → 로그인 페이지 이동 + 성공 토스트
  - 인증 링크 만료 → "링크가 만료되었습니다" + 재발송 안내
```

### 2.2 소셜 로그인 (Google OAuth2)

```
[로그인 페이지]
        │
        │ "Google로 시작하기" 클릭
        ▼
[Google OAuth2 동의 화면] (Google 제공)
        │
        ├── 동의 거부 → 로그인 페이지 복귀 + "로그인이 취소되었습니다"
        │
        └── 동의 완료
              │
              ▼
        [서버: Google 토큰 검증]
              │
              ├── 신규 사용자
              │     → 자동 회원 가입 (구글 계정 정보 기반)
              │     → 닉네임 설정 페이지 /signup/nickname
              │           │ 닉네임 입력 완료
              │           ▼
              │     [홈 / 상품 목록 페이지]
              │
              └── 기존 사용자
                    → JWT 발급
                    → 이전 페이지 또는 홈으로 이동
```

### 2.3 로그인 / 토큰 재발급 / 로그아웃

```
[로그인 페이지 /login]
        │
        │ 이메일 + 비밀번호 입력 후 "로그인"
        ▼
  [서버 인증]
        │
        ├── 실패 → "이메일 또는 비밀번호가 올바르지 않습니다" (인라인 에러)
        │          5회 연속 실패 → "잠시 후 다시 시도해주세요" (Rate Limit)
        │
        └── 성공
              → Access Token (30분) + Refresh Token (7일) 발급
              → Access Token: 메모리 저장
              → Refresh Token: HttpOnly Cookie 저장
              → 이전 페이지 또는 홈 이동

─────────────────────────────────────────
토큰 재발급 (자동, 사용자 비인지)
─────────────────────────────────────────
  API 요청 중 401 응답
        │
        ▼
  Refresh Token으로 재발급 요청
        │
        ├── 재발급 성공 → 원래 요청 재시도
        └── 재발급 실패 (만료/무효화) → 로그인 페이지 이동

─────────────────────────────────────────
로그아웃
─────────────────────────────────────────
  헤더 "로그아웃" 클릭
        │
        ▼
  서버: Refresh Token 무효화 (Redis에서 삭제)
  클라이언트: Access Token 메모리 삭제 + Cookie 삭제
        │
        ▼
  [로그인 페이지] + "로그아웃되었습니다" 토스트
```

## 3. 상품 플로우

### 3.1 상품 등록 (판매자)

```
[헤더 "상품 등록" 버튼]
        │
        │ 비로그인 → 로그인 페이지 리다이렉트
        │
        ▼
[상품 등록 페이지 /items/new]
  ┌─ Step 1: 기본 정보 ──────────────────────────────┐
  │  - 제목 입력                                      │
  │  - 카테고리 선택 (대분류 → 중분류)                │
  │  - 가격 입력                                      │
  │  - 거래 방식 선택 (직거래 / 택배 / 둘 다)         │
  │                                                   │
  │  [AI 설명 자동완성 버튼] ──▶ Claude API 호출      │
  │    로딩 스피너 표시 (스트리밍)                    │
  │    생성된 설명 텍스트 영역에 자동 채워짐           │
  │    사용자가 직접 수정 가능                        │
  │                                                   │
  │  [AI 가격 추천 버튼] ──▶ Claude API 호출          │
  │    추천 가격 범위 툴팁으로 표시                   │
  └───────────────────────────────────────────────────┘
        │
  ┌─ Step 2: 이미지 업로드 ──────────────────────────┐
  │  - 이미지 드래그 앤 드롭 / 파일 선택              │
  │  - 최대 10장, 파일당 5MB                          │
  │  - 첫 번째 이미지가 썸네일로 자동 지정            │
  │  - 순서 변경 (드래그)                             │
  └───────────────────────────────────────────────────┘
        │
        │ "등록하기" 클릭
        ▼
  [서버: 상품 저장 + AI 사기 패턴 감지 비동기 실행]
        │
        ├── 등록 성공
        │     → [상품 상세 페이지 /items/:id] + "상품이 등록되었습니다" 토스트
        │
        └── 등록 실패 (유효성 오류)
              → 해당 필드 인라인 에러 표시
```

### 3.2 상품 탐색 / 검색 (구매자)

```
[홈 / 상품 목록 페이지 /items]
        │
  ┌─────┴──────────────────────────────────┐
  │  필터 / 검색 영역                       │
  │  - 키워드 검색 (제목 + 설명)            │
  │  - 카테고리 필터 (대/중분류)            │
  │  - 가격 범위 (최소 ~ 최대)              │
  │  - 거래 방식 (직거래/택배/둘 다)        │
  │  - 거래 상태 (판매중/예약중/거래완료)   │
  │  - 정렬 (최신순 / 인기순)              │
  └─────┬──────────────────────────────────┘
        │
        ▼
  [상품 카드 목록] (커서 기반 무한 스크롤)
  - 썸네일, 제목, 가격, 거래 방식, 관심 수, 거래 상태
        │
        │ 카드 클릭
        ▼
  [상품 상세 페이지 /items/:id]
```

### 3.3 상품 상세 조회

```
[상품 상세 페이지 /items/:id]
  ┌───────────────────────────────────────┐
  │  이미지 갤러리 (스와이프)              │
  │  제목 / 가격 / 카테고리 / 거래 방식   │
  │  설명 텍스트                          │
  │  판매자 정보 (닉네임, 신뢰 점수, 등급) │
  │  거래 상태 뱃지                       │
  └───────────────────────────────────────┘
        │
        ├── [찜하기 버튼] ──▶ 관심 상품 토글 (로그인 필요)
        │
        ├── [채팅하기 버튼] ──▶ 채팅방 생성/이동 (로그인 필요)
        │                       (상품:구매자 쌍으로 유니크)
        │
        ├── [구매 요청 버튼] ──▶ 거래 요청 플로우 진입
        │   (상태가 SALE일 때만 활성화)
        │
        ├── [예약 슬롯 캘린더] ──▶ 예약 플로우 진입
        │   (판매자가 슬롯 등록한 경우 표시)
        │
        └── [본인 상품인 경우]
              ├── [수정하기] → /items/:id/edit
              ├── [끌어올리기] → 최대 1회/일
              ├── [숨기기] → 임시 비공개 처리
              └── [삭제하기] → 거래 진행 중이면 불가
```

## 4. 거래 플로우

### 4.1 거래 요청 ~ 완료 (구매자 시점)

```
[상품 상세 페이지] — 상태: SALE
        │
        │ "구매 요청" 클릭
        ▼
[구매 요청 모달]
  - 메시지 입력 (선택)
  - "요청 보내기" 클릭
        │
        ▼
  상태: PENDING (요청 대기)
  → 판매자에게 알림 발송
        │
        ├── 판매자 거절
        │     상태: REJECTED
        │     → 구매자에게 알림 발송
        │     → [거래 내역] 거절됨 표시
        │
        └── 판매자 수락
              상태: RESERVED (예약중)
              → 구매자에게 알림 발송
                    │
                    │ 거래 진행 (직거래 또는 택배)
                    │
                    │ "거래 완료 확인" 클릭
                    ▼
              상태: COMPLETED 대기 (구매자 확인 완료)
              → 판매자에게 "거래 완료 확인 요청" 알림
                    │
                    ├── 판매자도 확인
                    │     상태: COMPLETED (거래완료)
                    │     → 양측에 알림 발송
                    │     → 리뷰 작성 유도 팝업
                    │
                    └── 거래 취소 요청
                          → 취소 사유 입력 모달
                          상태: CANCELLED
                          → 상대방에게 알림 발송
```

### 4.2 거래 요청 ~ 완료 (판매자 시점)

```
[알림] "새 구매 요청이 왔습니다"
        │
        ▼
[거래 내역 /trades] 또는 알림 클릭
  - 구매자 정보, 메시지 확인
        │
        ├── "수락" 클릭
        │     상태: RESERVED
        │     → 구매자에게 수락 알림
        │     → 채팅으로 직거래 장소/방법 협의 유도
        │
        └── "거절" 클릭
              → 거절 사유 입력 (선택)
              상태: REJECTED
              → 구매자에게 거절 알림

─────────────────────────────────────────
거래 완료 확인 (판매자)
─────────────────────────────────────────
[알림] "구매자가 거래 완료 확인을 요청했습니다"
        │
        ▼
  "거래 완료 확인" 클릭
        │
        ▼
  양측 확인 완료 → 상태: COMPLETED
  신뢰 점수 +5점 (양측)
```

### 4.3 거래 상태 머신 전이

```
                     ┌──────────────────────────┐
                     │          SALE             │
                     │       (판매중)             │
                     └──────────┬───────────────┘
                                │ 구매자: 구매 요청
                                ▼
                     ┌──────────────────────────┐
                     │         PENDING           │
                     │       (요청 대기)          │
                     └──────┬─────────┬──────────┘
                            │         │
              판매자: 수락   │         │  판매자: 거절
                            ▼         ▼
               ┌──────────────┐  ┌──────────────┐
               │   RESERVED   │  │   REJECTED   │
               │   (예약중)    │  │  (거절됨)     │
               └──────┬───────┘  └──────────────┘
                      │                (종료)
         양측 완료 확인 │
                      ▼
               ┌──────────────┐
               │  COMPLETED   │
               │  (거래완료)   │
               └──────────────┘

  [SALE / PENDING / RESERVED] ──취소──▶ [CANCELLED] ──▶ (종료)
```

## 5. 예약 플로우

### 5.1 슬롯 등록 (판매자)

```
[마이페이지 > 예약 슬롯 관리 /profile/slots]
        │
        │ "슬롯 추가" 클릭
        ▼
[슬롯 등록 모달]
  - 날짜 선택 (오늘 ~ 30일 이내)
  - 시간 선택 (시작 시간 / 종료 시간)
  - 반복 설정 (선택): 매주 특정 요일
        │
        │ "등록" 클릭
        ▼
  [서버: 슬롯 저장]
        │
        ├── 성공 → 캘린더에 슬롯 표시 (초록색: 가용)
        └── 실패 (중복 시간) → "해당 시간에 이미 슬롯이 있습니다"

─────────────────────────────────────────
슬롯 삭제
─────────────────────────────────────────
  캘린더에서 슬롯 클릭
        │
        ├── 예약되지 않은 슬롯 → "삭제" 버튼 활성
        │     삭제 확인 → 슬롯 제거
        │
        └── 예약된 슬롯 → "삭제" 버튼 비활성
              "예약된 슬롯은 삭제할 수 없습니다" 안내
```

### 5.2 예약 요청 ~ 확정 (구매자)

```
[상품 상세 페이지] — 예약 슬롯 캘린더 섹션
        │
        │ 초록색(가용) 슬롯 클릭
        ▼
[예약 요청 모달]
  - 선택한 날짜/시간 확인
  - "예약 요청" 클릭
        │
        ▼
  [서버: 슬롯 임시 잠금 (Redis TTL 5분)]
  슬롯 상태: 노란색 (임시 잠금)
  → 판매자에게 "예약 요청 도착" 알림
  → 구매자에게 "5분 이내 판매자 확인을 기다리세요" 안내
        │
        ├── 판매자가 5분 내 수락
        │     [서버: 슬롯 확정 (DB 저장), 잠금 해제]
        │     슬롯 상태: 빨간색 (예약 확정)
        │     → 구매자에게 "예약 확정" 알림
        │     → 거래 상태: RESERVED
        │
        ├── 판매자가 5분 내 미응답
        │     [서버: TTL 만료 → 잠금 자동 해제]
        │     슬롯 상태: 초록색 (다시 가용)
        │     → 구매자에게 "예약이 만료되었습니다" 알림
        │
        └── 예약 취소 (예약 24시간 전까지)
              "예약 취소" 클릭 → 취소 확인 모달
              → 슬롯 다시 가용 상태로 변경
              → 상대방에게 취소 알림

─────────────────────────────────────────
예약 1시간 전 리마인더
─────────────────────────────────────────
  스케줄러가 예약 시간 1시간 전 감지
        │
        ▼
  양측 모두에게 SSE 알림 발송
  "1시간 후 직거래 예약이 있습니다"
```

## 7. 알림 플로우

```
[로그인 상태 유지 중 — 자동 SSE 연결]
  서버에서 이벤트 발생 시 실시간 푸시
        │
        ▼
[헤더 알림 아이콘 — 빨간 뱃지 (읽지 않은 수)]
        │
        │ 클릭
        ▼
[알림 드롭다운 / /notifications]
  - 최근 30일 알림 목록
  - 읽지 않은 알림: 하이라이트
  - 각 알림 클릭 → 해당 페이지 이동 + 읽음 처리
  - "전체 읽음" 버튼
        │
        │ SSE 연결 끊김 (탭 이동, 네트워크 오류 등)
        ▼
  자동 재연결 로직 (Exponential Backoff)
  재연결 성공 시 미수신 알림 HTTP Polling으로 보완
```

**알림 이벤트 → 이동 페이지 매핑:**

| 알림 이벤트 | 이동 페이지 |
|------------|------------|
| 새 채팅 메시지 | `/chats/:id` |
| 구매 요청 도착 | `/trades` (판매자) |
| 요청 수락 / 거절 | `/trades` (구매자) |
| 거래 완료 확인 요청 | `/trades/:id` |
| 거래 완료 | `/trades/:id` |
| 거래 취소 | `/trades/:id` |
| 예약 요청 도착 | `/profile/slots` |
| 예약 확정 / 취소 | `/items/:id` |
| 예약 1시간 전 리마인더 | `/items/:id` |
| 리뷰 등록 | `/profile/:userId` |
| 신고 처리 결과 | `/notifications` |
| 관심 상품 가격 인하 | `/items/:id` |

## 11. 페이지 전환 시나리오

### 시나리오 A — 신규 판매자 (박지훈): 첫 상품 등록

```
① 랜딩 페이지 (/)
    → "시작하기" 클릭
② 회원가입 (/signup)
    → 이메일/비밀번호/닉네임 입력
③ 이메일 인증 안내 페이지
    → 메일 링크 클릭
④ 로그인 페이지 (/login)
    → 로그인 완료
⑤ 상품 목록 (/items)
    → "상품 등록" 클릭
⑥ 상품 등록 (/items/new)
    → 제목/카테고리 입력 후 "AI 설명 자동완성" 사용
    → 이미지 업로드
    → "등록하기"
⑦ 상품 상세 (/items/:id)
    → "슬롯 추가"로 예약 슬롯 등록 안내
⑧ 슬롯 관리 (/profile/slots)
    → 직거래 가능 날짜/시간 등록
```

### 시나리오 B — 구매자 (이소연): 상품 탐색 → 예약 → 거래 완료

```
① 상품 목록 (/items)
    → 카테고리/가격 필터 적용
② 상품 상세 (/items/:id)
    → 판매자 신뢰 점수 / 리뷰 확인
    → 찜하기
    → "채팅하기" 클릭 (비로그인 → 로그인 이동)
③ 로그인 (/login) → 로그인 완료
    → 채팅방으로 자동 이동
④ 채팅방 (/chats/:id)
    → 가격 협의 후 "구매 요청" 버튼 클릭
⑤ 상품 상세 (/items/:id)
    → 예약 슬롯 캘린더에서 슬롯 선택 → 예약 요청
    → "5분 내 확인 대기" 안내
⑥ [알림] 예약 확정
    → 거래 내역 (/trades) 확인
⑦ 직거래 완료 후 "거래 완료 확인"
⑧ [알림] 거래 완료
⑨ 리뷰 작성 모달 → 별점 + 텍스트 제출
⑩ 판매자 프로필 (/profile/:id)
    → 작성한 리뷰 확인
```

### 시나리오 C — 관리자: 신고 처리

```
① 관리자 로그인 (/admin/login)
② 대시보드 (/admin)
    → "신고 3건 미처리" 알림 확인
③ 신고 처리 (/admin/reports)
    → 신고 목록 확인
    → 신고 상세 클릭
    → "AI 신고 요약" 버튼으로 내용 요약 생성
    → 처리 유형 선택 (경고 / 정지 / 기각)
    → "처리 완료"
④ 회원 관리 (/admin/users)
    → 제재 대상 회원 상세 조회
    → 상태 "7일 정지"로 변경
⑤ 대시보드 (/admin)
    → 처리 후 신고 수 감소 확인
```
