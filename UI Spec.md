## 1. 디자인 시스템

### 1.1 컬러

```
Primary   — green-600  (#16a34a)  브랜드 주색, 주요 CTA 버튼
Primary L — green-50   (#f0fdf4)  배경 강조, 카드 선택 상태
Secondary — blue-500   (#3b82f6)  링크, 보조 액션
Warning   — amber-500  (#f59e0b)  예약중, 대기 상태
Danger    — red-500    (#ef4444)  취소, 에러, 경고
Success   — green-500  (#22c55e)  완료, 성공 토스트
Neutral   — gray 계열  UI 기본 텍스트/보더/배경
```

**거래 상태 컬러 매핑:**

| 상태 | 뱃지 색 | Tailwind 클래스 |
|------|--------|----------------|
| SALE (판매중) | green | `bg-green-100 text-green-700` |
| PENDING (요청 대기) | amber | `bg-amber-100 text-amber-700` |
| RESERVED (예약중) | blue | `bg-blue-100 text-blue-700` |
| COMPLETED (거래완료) | gray | `bg-gray-100 text-gray-500` |
| CANCELLED (취소됨) | red | `bg-red-100 text-red-500` |
| REJECTED (거절됨) | red | `bg-red-100 text-red-500` |

**신뢰 등급 컬러 매핑:**

| 등급 | 아이콘 | 컬러 |
|------|--------|------|
| 씨앗 (SEED) | 🌱 | `text-gray-400` |
| 새싹 (SPROUT) | 🌿 | `text-green-500` |
| 열매 (FRUIT) | 🍎 | `text-orange-500` |
| 나무 (TREE) | 🌳 | `text-emerald-600` |

### 1.2 타이포그래피

**폰트:** Pretendard (한글 최적화 sans-serif, CDN 로드)

```
display  — text-2xl (24px) font-bold    페이지 타이틀
heading  — text-xl  (20px) font-semibold 섹션 제목
subhead  — text-lg  (18px) font-medium   카드 제목, 상품명
body     — text-base(16px) font-normal   본문, 설명
small    — text-sm  (14px) font-normal   메타 정보, 날짜
caption  — text-xs  (12px) font-normal   뱃지, 레이블
```

### 1.3 스페이싱 / 반응형 브레이크포인트

**Tailwind 기본 4px 단위 사용 (p-4 = 16px)**

| 이름 | px | 용도 |
|------|-----|------|
| `p-2` | 8 | 아이콘 버튼, 태그 내부 |
| `p-3` | 12 | 인풋, 작은 카드 |
| `p-4` | 16 | 카드, 섹션 기본 패딩 |
| `p-6` | 24 | 페이지 컨텐츠 패딩 |
| `gap-4` | 16 | 카드 그리드 간격 |
| `gap-6` | 24 | 섹션 간 간격 |

**반응형 브레이크포인트:**

| 이름 | 범위 | 레이아웃 |
|------|------|---------|
| mobile | < 640px | 단일 컬럼, 하단 네비게이션 |
| tablet | 640px ~ 1024px | 2컬럼 그리드 |
| desktop | ≥ 1024px | 사이드바 + 3~4컬럼 그리드 |

### 1.4 공통 컴포넌트

#### BaseButton

```
variant:
  primary   — bg-green-600 text-white hover:bg-green-700
  secondary — bg-white border border-gray-300 text-gray-700
  danger    — bg-red-500 text-white hover:bg-red-600
  ghost     — text-green-600 hover:bg-green-50

size:
  sm — text-sm px-3 py-1.5 rounded-md
  md — text-base px-4 py-2 rounded-lg   (기본)
  lg — text-lg px-6 py-3 rounded-lg

상태:
  loading  — 텍스트 숨김 + 스피너 중앙 표시, disabled

#### BaseInput

```
기본 구조:
  라벨 (text-sm text-gray-700 mb-1)

  normal  — border-gray-300
  focused — border-green-500 ring-2 ring-green-200
  error   — border-red-500 ring-2 ring-red-200
  success — border-green-500
  disabled— bg-gray-50 text-gray-400

#### BaseModal

```

│ backdrop: bg-black/50 fixed inset-0     │
│                                         │
│  ┌──────────────────────────────────┐   │
│  │ 모달 헤더                     ✕  │   │
│  ├──────────────────────────────────┤   │
│  │                                  │   │
│  │   모달 내용                      │   │

애니메이션: opacity + scale (200ms ease-out)
모바일: 하단 시트 (slide-up, rounded-t-2xl)
```

#### TradeStatusBadge

```
<span class="inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium {color}">
  {statusLabel}
</span>

statusLabel 매핑:
  SALE      → "판매중"
  PENDING   → "요청 대기"
  RESERVED  → "예약중"
  COMPLETED → "거래완료"
  CANCELLED → "취소됨"
  REJECTED  → "거절됨"
```

#### TrustScoreBadge

```
[🌿 새싹 · 65점]
  아이콘 + 등급명 + 점수 (text-xs)
  프로필 카드 / 상품 카드 판매자 정보에 표시
```

#### InfiniteScrollList

```
- Intersection Observer로 마지막 요소 감지
- 로딩 중: 하단 스켈레톤 카드 2장 표시
- 더 이상 없음: "마지막 상품입니다" 안내
- 에러: "다시 시도" 버튼
```

#### ToastNotification

```
위치: 화면 우하단 (mobile: 하단 중앙)
타입:
  success — bg-green-600  ✓ 아이콘
  error   — bg-red-500    ✕ 아이콘
  info    — bg-gray-800   ℹ 아이콘
  warning — bg-amber-500  ⚠ 아이콘

자동 사라짐: 3초 (error: 5초)
Stack: 최대 3개 동시 표시, 아래서 위로 쌓임
```

**ItemCard 컴포넌트 상세:**

```
┌──────────────────────────┐
│ [이미지 썸네일 1:1]       │
│                          │
│  [COMPLETED] 거래완료    │  ← 상태 오버레이 (반투명)
├──────────────────────────┤
│ 맥북 프로 M3 14인치      │  subhead
│ 2,500,000원              │  heading font-bold
│ 📦 직거래 · 노트북       │  caption text-gray-400
│ ♡ 23  · 서울 마포구      │  caption
└──────────────────────────┘

♡ 버튼: 로그인 시 토글 (즉시 optimistic update)
COMPLETED 상태: 이미지 위 반투명 회색 오버레이 + 텍스트
```

**검색창 동작:**
```
- 타이핑 시 300ms 디바운스 후 API 호출
- ESC: 검색 초기화
- 최근 검색어 5개 로컬스토리지 저장 (포커스 시 드롭다운)
- 검색 결과 0건: "검색 결과가 없습니다" + 필터 초기화 제안
```

## 4. 공통 인터랙션 패턴

### 4.1 낙관적 업데이트 (Optimistic Update)

```
찜하기 토글:
  1. 클릭 즉시 UI 변경 (♡ → ♥, wish_count +1)
  2. API 요청 전송
  3. 실패 시: 원래 상태로 롤백 + 에러 토스트

읽음 처리:
  1. 클릭 즉시 뱃지 감소
  2. 백그라운드 API 호출
```

### 4.2 무한 스크롤

```
- 마지막 카드 도달 80% 시점에서 다음 페이지 프리페치
- 로딩 중: 하단 SkeletonCard 2장 표시
- 네트워크 오류: "다시 시도" 버튼 표시
- 마지막 페이지: "마지막 상품입니다" 텍스트
```

### 4.3 폼 유효성 검사 타이밍

```
- 이메일/닉네임 중복: 포커스 아웃 + 500ms 디바운스
- 비밀번호 강도: 타이핑 중 실시간
- 나머지 필드: 폼 제출 시 (또는 포커스 아웃)
- 제출 실패 시: 첫 번째 에러 필드로 자동 스크롤
```

### 4.4 페이지 이탈 방지

```
상품 등록/수정 폼에서 뒤로가기 또는 링크 클릭 시:
  "작성 중인 내용이 있습니다. 나가시겠습니까?" 확인 모달
  (Vue Router의 beforeRouteLeave 훅)
```

적용 대상: 상품 목록, 상품 상세, 채팅 목록, 프로필

## 5. 상태별 UI 처리

### 5.1 로딩 상태

| 상황 | 처리 방식 |
|------|----------|
| 첫 페이지 로드 | 스켈레톤 UI |
| 버튼 클릭 (API 대기) | 버튼 내 스피너 + disabled |
| AI 생성 중 | 스트리밍 텍스트 + 커서 깜빡임 |
| 이미지 업로드 | 개별 이미지 프로그레스 오버레이 |
| 페이지 전환 | 상단 1px 그린 프로그레스바 (NProgress 스타일) |

### 5.2 에러 상태

| 에러 유형 | 표시 방식 |
|----------|----------|
| 폼 유효성 오류 | 필드 하단 인라인 에러 텍스트 (red-500) |
| API 4xx 에러 | 에러 토스트 (액션 불가 이유 명시) |
| API 5xx 에러 | 에러 토스트 "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요." |
| 페이지 로드 실패 | 전체 화면 에러 UI + 새로고침 버튼 |
| 404 | 커스텀 404 페이지 |
| 401 (세션 만료) | "로그인이 필요합니다" 토스트 + 로그인 페이지 이동 |

### 5.3 빈 상태 (Empty State)

| 화면 | 빈 상태 메시지 |
|------|--------------|
| 상품 목록 (검색 결과 없음) | 🔍 "검색 결과가 없습니다" + 필터 초기화 버튼 |
| 거래 내역 없음 | 🛒 "아직 거래 내역이 없어요" + 상품 보러가기 버튼 |
| 채팅 목록 없음 | 💬 "아직 채팅이 없어요" |
| 알림 없음 | 🔔 "새로운 알림이 없습니다" |
| 관심 상품 없음 | ♡ "관심 상품을 등록해보세요" + 상품 탐색 버튼 |
| 리뷰 없음 | ⭐ "아직 리뷰가 없습니다" |

### 5.4 권한 / 접근 제한

```
비로그인 상태에서 인증 필요 기능 접근 시:
  - 채팅하기 / 구매요청 / 찜하기 클릭
    → "로그인이 필요한 서비스입니다" 모달
    → [로그인하기] / [회원가입하기] / [취소] 버튼

정지된 계정 로그인 시:
  → "계정이 정지되었습니다. (2026-06-01까지)" 에러 메시지

이메일 미인증 상태:
  → 헤더에 노란 배너 "이메일 인증을 완료해주세요. [인증 메일 재발송]"
  → 상품 등록, 구매 요청 등 주요 기능 접근 시 인증 안내 모달
```
